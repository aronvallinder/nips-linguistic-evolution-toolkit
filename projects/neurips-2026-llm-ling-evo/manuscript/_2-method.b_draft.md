## 3.1 Overview

Two LLM agents interact over repeated rounds of a **Trust (Investment) Game** [@berg1995], optionally paired with a **myth-writing task**. Each experimental run fixes a (model, persona, task-order, noise-condition) tuple and produces a canonical simulation-state JSON logging every agent turn, action, payoff, and generated text. All experiments are configuration-driven via a single YAML file, so each condition corresponds to a named `experiment_set` and is reproducible from the saved state.

## 3.2 Game

We use a role-swapping repeated Trust Game. Each round $t \in \{1, \dots, T\}$ has two sequential moves:

1. The **investor** receives an endowment $E = \$5$ and chooses an amount $s_t \in [0, E]$ to send to the **trustee**.
2. The trustee receives $m \cdot s_t$ (multiplier $m = 3$) and chooses an amount $r_t \in [0, m \cdot s_t]$ to return.

Per-round payoffs are $\pi^{I}_t = E - s_t + r_t$ for the investor and $\pi^{T}_t = m s_t - r_t$ for the trustee, computed from the agents' actual choices. Cumulative balances $B^{I}_T$, $B^{T}_T$ are the primary behavioural outcome. **Roles alternate every round** (agent 1 = investor on odd rounds, trustee on even), so a single agent's cumulative score reflects behaviour in both roles rather than a fixed position.

Agents receive the full game rules in the system prompt (both roles, payoff formulas, response schema). On round 1 the investor sees only the rules. From round 2 onward, each agent is shown the **previous round's** $s_{t-1}$, $r_{t-1}$, derived percentages, own per-round payoff, and *its own perceived* cumulative balance $\tilde{B}_t$. The latter qualifier is important once noise is in play (§3.4): under noisy conditions the figure shown to each agent is the *communicated* balance — accumulated from the values reported to it round by round — which can drift from the ground-truth balance used for analysis. Memory truncation in the agent's message buffer (`memory_capacity = 3`) means each agent effectively sees the last three exchanges plus the system prompt; combined with the prompt's explicit summary of the *previous* round, this gives a memory-1 observability structure for game state and a memory-3 observability structure for raw conversation context. All prompts (system, round-1 / later-round investor, round-1 / later-round trustee, myth) are externalised to YAML templates so the wording is reproducible across runs. Responses are extracted by regex from a `{'send': <amount>}` or `{'return': <amount>}` JSON payload; malformed responses raise and abort the run rather than being silently coerced.

Default run length in the experiments reported here is $T = 10$ rounds (the value used across all noise-condition experiment sets); larger horizons (up to $T = 50$) appear in earlier non-noise pilots.

## 3.3 Myth-writing task

In myth conditions, each round also contains a creative-writing turn in which each agent writes a 200-word myth. On round 1 the prompt is minimal: `Write a myth about {myth_topic}. Write 200 words.`. The topic is configurable per run from a small fixed inventory — `anything` (deliberately unconstrained), `the rise and spread of moralizing supernatural agents` (a prosocial-religion / cultural-evolution theme), and `a trickster who exploits others and the price that follows` (a defection-motif theme). All noise-condition experiment sets reported here use the unconstrained `anything` topic, so the myth content is endogenously chosen rather than thematically primed. From round 2 onward, the prompt includes both (i) the agent's own myth from the previous round and (ii) the other agent's myth from the previous round, with the instruction to "use your previous myth as inspiration, but adapt it in your own way." This yields an **iterated-learning-style transmission chain** [@kirby2008] inside a dyad.

The game and myth tasks share an agent's message buffer but interact only through that buffer: the trust-game prompts (§3.2) do not directly inject the partner's myth as game context. Cross-task influence on game play therefore flows through (i) the agent's own previously-written myth, which sits in its context window when it next plays the game, and (ii) the partner's previous myth, which the agent has seen quoted inside its *own* round-$\geq\!2$ myth-writing prompt.

## 3.4 Experimental conditions

Conditions factor across four dimensions. A guiding principle for the noise designs is **asymmetric perspective**: each agent sees ground-truth information about its *own* actions (what it sent or returned) but potentially noised information about the *other* agent's intentions (what the other appeared to send or return). The game's payoff bookkeeping is run on actual amounts; only the inter-agent signal is corrupted. The simulation maintains two parallel ledgers — actual `balances` (used for analysis) and `balances_communicated` (what each agent perceives) — that diverge over noisy rounds.

1. **Task order** $\in \{\text{game only},\ \text{myth only},\ \text{game} \to \text{myth},\ \text{myth} \to \text{game}\}$. Game-only and myth-only serve as baselines isolating each task; the two ordered conditions test whether myth-writing shapes subsequent game behaviour (myth→game) and whether game outcomes shape subsequent myth content (game→myth).
2. **Noise condition.** Six configured presets, paired into informed / uninformed variants of three underlying noise mechanisms:
    - *No-noise (`default`)*: communicated amounts equal actual amounts. Baseline.
    - *Bidirectional uniform ±$1 (`noisy_bidirectional`, `noisy_bidirectional_informed`)*: the value of *sent* shown to the trustee, and the value of *returned* shown to the investor, each receive an additive perturbation drawn independently from $\mathcal{U}(-1, +1)$ (clamped to the action's valid range). Designed to perturb ceiling-locked or floor-locked dyads without changing expected payoffs.
    - *Negative-only uniform $\le 5$ (`noisy_negative_5`, `noisy_negative_5_informed`)*: the perturbation is drawn from $\mathcal{U}(-5, 0)$ — the partner systematically appears *less* generous than they actually were, by up to the full endowment. Targets ceiling-locked models (Gemini, Claude) by introducing apparent betrayal.
    - *Bootstrap (`noisy_bootstrap_cooperation`, `noisy_bootstrap_cooperation_informed`)*: the trustee's return is replaced (with probability 1.0, every round) by its maximum possible value when reported back to the investor. The investor sees apparent full reciprocation regardless of what was actually returned. Targets floor-locked models (GPT-5-Nano) by manufacturing the appearance of cooperation.
    - In each *_informed* variant, a single sentence is appended to the system prompt: "There may be communication noise in this game. The amounts you see may differ from what was actually sent/returned. Can you filter out the noise?". Otherwise the agents are unaware of the noise.
3. **Asymmetric naming (`other_player_names`).** A separate framing manipulation: the OTHER player's role can be swapped in the prompts for a mythological label (`prosocial`: Prometheus / Orpheus; `trickster`: Coyote / Anansi; `exchange`: Persephone / Yggdrasil; `neutral_myth`: The Wanderer / The Guardian) while the agent's own role stays generic. The active noise-experiment sets all use the `default` neutral naming; mythological-naming runs are infrastructure-ready but out of scope for the noise-effect results reported here `[CHECK — confirm none of the headline results use non-default naming]`.
4. **Persona** $\in \{\text{neutral},\ \text{altruistic},\ \text{selfish}\}$. Personas are injected as a one-sentence addition to the system prompt. All noise experiments reported here use *neutral* only.
5. **Model.** Agents in a dyad always share a base model. The three primary models, chosen to span the apparent sociality spectrum: Anthropic `claude-sonnet-4.5` (ceiling-locked), Google `gemini-3.1-pro-preview` (ceiling-locked), and OpenAI `gpt-5-nano` (floor-locked). Earlier non-noise pilots covering Llama-3.1-8B-Instruct, GPT-4o, Claude Haiku, DeepSeek-V3 and others are reported in the appendix.

Each cell of the design is run with $N = 10$ independent seeds (15 for the GPT-5-Nano negative-noise cell); all runs use temperature $0.8$, `memory_capacity = 3`, and identical system / game / myth prompt templates pulled from a single YAML config.

## 3.5 Measures

**Behavioural.** Per-round $s_t$, $r_t$, role-specific payoffs, and cumulative balances. Summary statistics: mean send rate, mean return rate, cumulative reward by role, and variance across seeds. Condition contrasts are computed as **differences of medians** across seeds rather than means, to avoid averaging over qualitatively different trajectories (e.g., lock-in vs. oscillation) that share the same mean.

**Linguistic.** For each myth we compute:

- **Cooperativity word counts.** Frequency of a hand-curated cooperativity lexicon (cooperate, trust, share, together, …) with standard stopword removal and simple $>2$-character filtering. Dictionary-based and case-insensitive; we flag (and do not adjust for) the known limitations — no stemming, no negation handling.
- **LIWC pronoun ratios.** We report the *we/I ratio* ($W / (I + 1)$) and the *we-proportion* ($W / (I + W + Y + T)$), following standard LIWC conventions [@tausczik2010], as a coarse proxy for group-identity framing.
- **Pairwise myth similarity.** Two methods applied in parallel: (a) **sentence-embedding cosine similarity** using `all-mpnet-base-v2`; (b) **LLM-as-judge** on five dimensions (thematic, character, narrative, symbolic, conceptual) on a 0–10 scale, averaged, using Llama-3.3-70B via the Together API `[CHECK judge model]`. Both methods are applied (i) *within-agent* across consecutive rounds (myth drift) and (ii) *between-agent* within a round (linguistic convergence).
- **Sentence-structure / n-gram measures.** Using spaCy tokenisation we compute POS-sequence n-gram distributions per myth, a sentence-level *innovation rate* (fraction of POS patterns not previously observed), and distributional entropy. These target the structural side of linguistic convergence that word-level measures miss.
- **Keyword chain evolution.** Per-round top-$k$ keyword extraction and cross-round tracking, to surface emergent vocabulary and neologisms (e.g., the "kyrexladilokrater"-style coinages observed in pilot runs).
- **Lag correlation.** Pearson correlation between agent $A$'s cooperativity-word score at round $t$ and agent $B$'s at round $t+1$, as an index of mimicry / convergence dynamics.

## 3.6 Statistical inference

Within each noise × task-order × model cell we compare game-only, myth-game, and game-myth task orderings on (i) cumulative reward and (ii) linguistic convergence. Our pre-specified primary contrast is the **median delta in cumulative reward between the myth-present and game-only conditions**, computed within each noise condition. We also report variance across seeds as a second-order outcome, since one hypothesis for the role of myth is *strategy consolidation* (lower across-seed variance) rather than *cooperation lift* (higher mean). Non-parametric bootstrap confidence intervals are reported on all cell-level estimates `[CHECK — confirm bootstrap procedure with Ivar]`.

## 3.7 Simulation flow

A single batch run resolves an `experiment_set` from `noise/experiments_noisy.yaml` into the cartesian product of {model} × {prompt template} × {persona} × {task order} × {myth topic} × {`game_params` preset}, multiplied by `num_runs` independent seeds. Each cell of that product is one independent simulation, executed sequentially or via `ProcessPoolExecutor` workers (`--workers N`). Within a cell, both agents share the same persona and the same base model, so the manipulation is between-dyad rather than within-dyad.

**Per-run setup.** The engine instantiates a `TrustGameNoisy` (with $E$, $m$, the system-prompt template, the persona dict, the four round-specific prompt templates, the `noise_config`, and the `other_player_names` set) and a `MythWriter` (with the resolved myth topic and the round-1 / later-round prompt templates). It then creates `num_agents = 2` `Agent` objects, each holding its own model id, temperature, memory capacity, and message buffer. The game-rules system prompt — including both investor and trustee roles, the payoff formulas, the response schema, any persona one-liner, and (in noise-informed conditions) a single sentence flagging that "amounts you see may differ from what was actually sent/returned" — is written into each agent's buffer as the initial `system` message. The message buffer is shared across game and myth turns within the run. A per-run `.log` file is opened to capture every prompt, response, reasoning trace, and token-usage record verbatim.

**Per-round loop.** For each round $t \in \{1, \dots, T\}$ the engine performs the following steps in order:

1. **Role assignment.** `TrustGameNoisy.get_roles_for_round(t)` returns investor/trustee for $t$ (alternating each round); `get_move_order(t)` fixes the move order to *investor first, then trustee*.
2. **Round-record initialisation.** A per-round dict is appended to `conversation_history` with all canonical fields pre-nulled — `sent`, `sent_communicated`, `received`, `received_communicated`, `returned`, `returned_communicated`, `investor_payoff`, `trustee_payoff`, `balances`, `balances_communicated`, `actions`, `myths`, `noise_applied`, `other_player_names`, `game_responses`, `myth_responses` — so partial state remains addressable on failure.
3. **Tasks executed in `task_order`.** For each task name in the per-cell `task_order` list:
    - **Game.** Sequential. The investor is prompted (round 1: just the round number, role, endowment; round $t \geq 2$: a templated summary of the previous round in the agent's *perceived* frame — what the partner appeared to send, what the agent itself returned, the agent's own perceived per-round payoff, and the agent's own perceived cumulative balance). The investor responds; the *actual* send amount is regex-extracted from the `{'send': <amount>}` JSON payload and stashed in `game_data["pending_sent"]`. *Noise application point #1* fires here: `_apply_noise` is called on `pending_sent` and the result stored as `pending_sent_communicated`, which is what the trustee's prompt will quote. The trustee is then prompted with the *communicated* amount, responds, and the engine calls `process_turn`, which (i) re-extracts both actual amounts, (ii) fires *noise application point #2* on the trustee's returned amount, (iii) computes payoffs from the **actual** values, (iv) updates both `balances` (ground truth) and `balances_communicated` (per-agent perceived totals), (v) increments `noise_stats`, and (vi) fills both actual and communicated fields in the round record. If a mythological `other_player_names` set is configured, the partner's role label in every prompt above is swapped accordingly (e.g. the trustee sees "Prometheus sent you $X" instead of "the investor sent you $X"); the agent's own role label stays generic.
    - **Myth.** Parallel. Both agents' myth prompts are constructed (round 1: minimal `Write a myth about {myth_topic}. Write 200 words.`; round $t \geq 2$: a template containing *the agent's own previous myth* and *the other agent's previous myth*, with the instruction "use your previous myth as inspiration, but adapt it in your own way"). The two LLM calls are dispatched concurrently via a `ThreadPoolExecutor`. A single retry with a 1 s back-off is attempted on failure before propagating; on the second failure the exception bubbles up to the engine's outer handler. Both myth contents are written into the round record under `myths`, with the full structured response (content + reasoning trace + token usage) under `myth_responses`.
4. **Per-round persistence.** A lightweight `.results.json` (everything except agent message buffers) is written after every successful round. Every `checkpoint_every = 10` rounds, a full `.checkpoint.json` (including message buffers) is written for crash-recovery. On any uncaught exception, both the `.results.json` and an `.error.json` checkpoint are flushed before the engine re-raises. Resumption from `resume_from = <checkpoint>` rebuilds the full state — including each agent's message history — and continues from the next round.

**Cross-task influence.** Within a run, every game turn and every myth turn is appended to the *same* per-agent message buffer. The myth-writing prompt explicitly cites the partner's previous myth. The trust-game prompt does **not** inject the partner's myth — cross-task influence on game play therefore flows only through (i) the agent's own myth, which sits in its context window when it next plays the game, and (ii) any persistence the model carries from having seen the partner's myth quoted inside its own previous myth-writing prompt.

**Memory truncation.** Before each LLM call, an agent's buffer is truncated to `memory_capacity × 2 + 1` messages (system prompt + the most recent `memory_capacity` user/assistant exchanges). This is shared across game and myth turns, so a high myth cadence shortens the *game* horizon visible to the model.

**Failure handling.** Malformed game responses (where the regex cannot extract a numeric `send` / `return`) raise `ValueError` and abort the run rather than being silently clamped or zeroed; missing prior myths in round-$\geq\!2$ myth prompts likewise raise. Myth-writing receives a single retry; game responses do not. Partial / truncated runs are surfaced (with their `.error.json` checkpoint) rather than silently analysed.

**Persisted state.** At the end of each run, `SimulationData.save_state` writes a single JSON containing: (a) `conversation_history` — the list of per-round records (roles, both ledgers of sent/received/returned, both payoffs, both balances, actions, myths, structured `game_responses` and `myth_responses` with reasoning + token usage, `noise_applied`, `other_player_names`); (b) `game_data` — final actual and communicated balances, accumulated `noise_stats`, trailing pending values; (c) `task_order`; (d) `run_metadata` — model, temperature, num_turns, num_agents, memory_capacity, myth_topic_id, myth_topic, `game_params_name`, `noise_config`, `other_player_names`; and (e) the full per-agent message buffers, including any reasoning traces returned by the model. All downstream behavioural and linguistic analyses (§3.5) read from this single artefact.

## 3.8 Implementation

All noise experiments are driven by a single `noise/experiments_noisy.yaml` and executed by a common simulation engine that is agnostic to the game type; the Trust Game (with optional noise and asymmetric naming), the myth-writing task, the persona system, and task-order sequencing are all independent modules. Each run produces a deterministic, re-analysable simulation-state JSON together with a verbatim `.log` of every prompt, response, reasoning trace, and token-usage record. Code and configurations are open-sourced at [github.com/ivarfresh/nips-linguistic-evolution-toolkit](https://github.com/ivarfresh/nips-linguistic-evolution-toolkit). Model calls are routed via OpenRouter; LLM-as-judge similarity calls use the Together API. Compute for the full-factorial runs was supported by a CAIF compute grant.
